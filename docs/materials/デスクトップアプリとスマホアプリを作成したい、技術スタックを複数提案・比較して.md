## デスクトップ × スマホ対応 技術スタック比較

---

### 🥇 Option 1 — **Flutter（Dart）**
> コード共有率が最も高いクロスプラットフォーム

| 項目 | 評価 |
|------|------|
| 対応プラットフォーム | iOS / Android / Windows / macOS / Linux / Web |
| コード共有率 | ★★★★★（90%以上） |
| パフォーマンス | ★★★★☆（ネイティブに近い） |
| 学習コスト | ★★★☆☆（Dart言語の習得が必要） |
| 開発速度 | ★★★★☆ |
| エコシステム | ★★★★☆（Google製、急成長中） |

**向いているケース：** 1つのコードベースで全プラットフォームを一気に出したい場合。スタートアップ・個人開発に最適。

---

### 🥈 Option 2 — **React Native（JS/TS）+ Electron**
> Web開発者が最も入りやすい構成

| 項目 | 評価 |
|------|------|
| 対応プラットフォーム | iOS / Android（RN）+ Windows / macOS（Electron） |
| コード共有率 | ★★★☆☆（モバイルとデスクトップで別フレームワーク） |
| パフォーマンス | ★★★☆☆（Electronはメモリ消費大） |
| 学習コスト | ★★★★★（React経験者はほぼゼロ） |
| 開発速度 | ★★★★☆ |
| エコシステム | ★★★★★（最大規模のOSSコミュニティ） |

**向いているケース：** すでにReact/Web開発者がいるチーム。採用しやすく、外注もしやすい。

---

### 🥉 Option 3 — **Kotlin Multiplatform（KMP）**
> ネイティブ品質を妥協せず、ロジックだけ共有

| 項目 | 評価 |
|------|------|
| 対応プラットフォーム | iOS / Android / Desktop（JVM） |
| コード共有率 | ★★★☆☆（UIは各プラットフォームネイティブ） |
| パフォーマンス | ★★★★★（完全ネイティブ） |
| 学習コスト | ★★☆☆☆（Kotlin + Swift両方必要） |
| 開発速度 | ★★☆☆☆（UIを2重実装） |
| エコシステム | ★★★☆☆（成長中だが若い） |

**向いているケース：** ゲームや高パフォーマンスが必須なアプリ。チームが大きく、iOS/Androidエンジニアがいる場合。

---

### ⚡ Option 4 — **Tauri（Rust + React/Vue）+ React Native**
> 最新鋭・軽量デスクトップ × モバイル

| 項目 | 評価 |
|------|------|
| 対応プラットフォーム | Windows / macOS / Linux（Tauri）+ iOS / Android（RN） |
| コード共有率 | ★★★★☆（UIロジックをTS/JSで共有） |
| パフォーマンス | ★★★★★（Electronの1/10サイズ） |
| 学習コスト | ★★☆☆☆（Rustのバックエンドが難しい） |
| 開発速度 | ★★★☆☆ |
| エコシステム | ★★★☆☆（急成長中） |

**向いているケース：** デスクトップアプリの軽量性・セキュリティを重視する場合。Electronが重すぎると感じているチーム。

---

## まとめ：どれを選ぶべき？

```
個人開発 / スタートアップ  → Flutter 一択に近い
Web系チームがいる         → React Native + Electron
品質・パフォーマンス最優先 → KMP（ネイティブ）
デスクトップを軽くしたい   → Tauri + React Native
```
